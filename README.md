# Better Link
> 让 Obsidian 的标题链接（Header Links）从此坚不可摧。

在知识管理中，我们常常面临一个两难境地：过度原子化（Atomic Notes）会导致信息碎片化，难以维护；而保持长篇笔记时，引用的 `[[文件名#一级标题#二级标题]]` 链接又极其脆弱。

当你重命名标题、将内容剪切到新文件、或调整标题层级时，Obsidian 原生的链接追踪往往会失效。**Betterlink** 为此而生。

## ✨ 核心特性

### 1. 全自动重构修复
当你对笔记内容进行**重命名**或**剪切/粘贴（Refactoring）**时，Betterlink 会在后台静默工作，自动识别受影响的链接并完成修复。
- **跨文件移动：** 将一个标题及其下属内容剪切到另一个文件，所有指向该标题的链接会自动更新。
- **层级自动识别：** 能够自动识别剪切后的父级标题环境，确保 `[[#标题]]` 这种索引路径始终准确。

### 2. 深度标题追踪
不再局限于简单的文件名修改。Betterlink 深入处理 `[[Note#Header1#Header2]]` 这种多级索引。无论你如何移动或嵌套这些“块”，链接都不会断裂。

### 3. 高性能的大规模修复
针对拥有数千个链接的大型仓库进行了优化。即使在进行大规模的内容重组时，Betterlink 也能保持流畅的响应速度，不会导致编辑器卡顿。

### 4. 智能感知
- 自动识别由于剪切导致的内容结构变化。
- 智能处理标题重名冲突。

---

## 🚀 解决的痛点

| 场景 | Obsidian 原生表现 | 使用 Betterlink |
| :--- | :--- | :--- |
| **重命名标题** | 部分支持（有时失效） | **完美支持**，同步更新所有引用 |
| **剪切内容到新文件** | 链接变红（断开） | **自动修复**，将链接更新为新路径 |
| **改变标题层级** | 路径 `[[#A#B]]` 失效 | **动态更新**，自动计算新层级路径 |
| **大规模重组** | 手动一个个修复 | **秒级全自动处理** |

- **重度引用用户：** 笔记中大量使用 `#标题`  进行精准定位。
